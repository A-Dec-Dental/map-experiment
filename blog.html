Building an Interactive Global Map with AI

How we used Claude Code to create a pure HTML/CSS map component — including the hilarious missteps along the way.

We needed an interactive "Global Locations" map for our About Us page. Something modern, responsive, and embeddable in our CMS without JavaScript dependencies. Instead of hand-coding the whole thing, we built it collaboratively with Claude Code — Anthropic's AI coding assistant. Here's how it went, warts and all.

Starting with a Static Image

The starting point was a static map image already on our site — five A-dec locations pinned to a world map with colored badges. It looked fine, but it wasn't responsive, wasn't accessible, and couldn't be easily updated.

The goal: recreate it as a pure HTML/CSS component that could drop into any rich text editor. The requirements were straightforward:

  • Modern, clean design with subtle animation
  • Fully responsive across mobile, tablet, and desktop
  • Accessible markup
  • No JavaScript dependencies
  • Five locations: Newberg (HQ), Nashville, Nuneaton, Paris, and Sydney

The Hilarious First Attempt

Claude's first move was ambitious: hand-draw the entire world map as SVG paths from scratch. No reference file, no tracing — just raw coordinate guesses for every continent.

The result was... something.

North America looks like a deflated balloon. Africa is a teardrop. Australia is a shapeless oval. It was genuinely one of the funniest things I've seen an AI produce.

Lesson learned: AI-generated SVG map paths are not the way to go. A real map asset is essential — no matter how confident the AI sounds about "simplified continent outlines."

Sourcing a Real Map

We needed an actual SVG world map file. Claude researched public domain options (Wikimedia Commons, FreeSVG, GitHub repos), but the files were either too detailed, had country borders we didn't want, or were too large.

I ended up providing my own World.svg asset — but at 1.5 MB, it was too large for Claude to read or inline. The solution was elegant: reference it as a standard <img> tag and use a CSS filter to control the color:

This meant the SVG file size didn't matter at all. It loads normally as an image, and CSS handles the rest.

The Second Attempt

With a real map in place, Claude produced something that actually resembled Earth this time. Solid white map, colored badge labels, pulsing dots — the right visual direction. But the pins were still in the wrong spots, and the design felt safe. It was a slightly fancier version of what we already had.

Two things needed to happen: fix the pin positions, and push the design further. I asked Claude to take some creative liberty with the next version.

Updated Look and Feel

This is where things got interesting. Claude came back with a completely different aesthetic:

  • Ghost map — SVG dropped to 12% opacity, creating a subtle backdrop
  • Split typography — City name (bold), country (light), and type badge as separate visual layers
  • Radar-style ping animation — Double ripple rings on each dot with staggered timing
  • Labels always visible — Staggered fade-in on page load instead of hover-to-reveal

The vibe shifted from "corporate infographic" to something much more editorial and modern. Only one problem — the pins were nowhere near the right locations.

The Pin Placement Saga

This was the most entertaining debugging session. Claude tried three approaches to place the pins correctly:

Attempt 1: Math

Claude calculated pixel positions from latitude/longitude coordinates, assuming the SVG used an equirectangular projection. It didn't. Oregon ended up in Canada.

Attempt 2: Browser Console Script

Claude suggested pasting a JavaScript snippet into the browser console that would log click coordinates. It didn't work in my environment.

Attempt 3: Built-In Click Helper

The winning approach: Claude embedded a coordinate tool directly into the page. Click anywhere on the map, get a red dot and the exact CSS percentage position logged to a green overlay.

I clicked each city one at a time and fed the coordinates back. We did it iteratively — place one pin, verify, move to the next. The final positions:

Takeaway: When math fails and console scripts don't cooperate, build the debugging tool into the product itself. The human-in-the-loop approach of click-and-report was faster than any automated solution.

Fixing Label Collisions

With pins in the right spots, the next problem was obvious: Nuneaton and Paris are geographically close, and their labels were overlapping.

We tried connector lines (vertical, L-shaped) but they looked messy. The final solution was simple: remove all connector lines, move the UK label above and to the right of its dot, and push the Paris label further right. Clean separation, no visual clutter.

Then came the pixel-level tuning — a back-and-forth of "move Sydney up 10px," "Nashville down 5," "mirror the UK label horizontally." The kind of fine-tuning that only makes sense when you're looking at it live.

Polish & Production

The map was too tall for a webpage section — lots of empty Arctic space at the top. The fix: wrap the image and pins in a shared inner container, shift it upward with margin-top: -10%, and clip with overflow: hidden on the outer wrapper. Gradient fades on both edges blend seamlessly into the dark blue background.

The key insight was that the image and pins must move together. An earlier attempt moved only the image, which broke every pin position.

For the final production version, we stripped out everything interactive — hover tooltips, cursor changes, focus states, the coordinate helper. All CSS was scoped under a single .adec-map class with prefixed keyframe names, making it safe to paste into any CMS without style conflicts.

The whole process took about an hour. The biggest time sinks were pin placement and label collision — problems that required human eyes and iterative feedback. The AI handled the CSS architecture, responsive breakpoints, animation timing, and accessibility markup without much guidance.

Would I do it again this way? Absolutely. The collaboration model of "AI writes, human verifies visually, feeds corrections back" worked surprisingly well for a spatial layout problem. Just don't let it draw continents.
